Lemmings is a single Go binary with no external runtime dependencies. This page walks you through installation, your first load test, and how to read the output Lemmings produces. By the end, you will have run a real simulation and know what the numbers mean.

## Prerequisites

- Go 1.23 or later installed (`go version` to check)
- A running web application to test (local or staging)

## Install

<Steps>
  <Step title="Install the binary">
    Install Lemmings directly from source using `go install`:

    ```bash
    go install github.com/andreimerlescu/lemmings@latest
    ```

    This downloads, compiles, and places the `lemmings` binary in your `$GOPATH/bin` (typically `~/go/bin`).
  </Step>
  <Step title="Verify the installation">
    Check that the binary is available on your `$PATH`:

    ```bash
    which lemmings
    ```

    If `lemmings` is not found, ensure `~/go/bin` is in your `$PATH`:

    ```bash
    export PATH="$PATH:$(go env GOPATH)/bin"
    ```
  </Step>
</Steps>

## Run your first load test

<Steps>
  <Step title="Start a basic simulation">
    Run Lemmings against a local server. This command launches 10 terrain groups with 10 lemmings each (100 total) for 30 seconds:

    ```bash
    lemmings -hit http://localhost:8080/ -terrain 10 -pack 10 -until 30s
    ```

    Lemmings will:
    1. Index your application's URLs from its sitemap (or crawl if no sitemap exists)
    2. Print a boot summary
    3. Print a one-time dashboard token
    4. Begin ramping lemmings online
  </Step>
  <Step title="Read the boot summary">
    Before any lemming moves, Lemmings prints a summary of what is about to happen:

    ```text
    lemmings v1.0.0
    ─────────────────────────────────────────
      target:        http://localhost:8080/

      terrain:       10 groups
      pack:          10 lemmings per terrain
      total:         100 lemmings

      until:         30s per lemming
      ramp:          5m0s to full concurrency
      est. duration: ~5m30s wall clock

      limit:         100 goroutines
      save-to:
        → .
      dashboard:     http://localhost:4000
    ─────────────────────────────────────────
      lemmings are gathering...
    ```

    The `est. duration` is `ramp + until`. With the default 5-minute ramp and 30-second lifespan, the wall clock is approximately 5m30s. Reduce `-ramp` for faster tests.
  </Step>
  <Step title="Watch the live ticker">
    Once lemmings are running, the terminal updates every second:

    ```text
    [0m45s] terrains: 2 | alive: 20 | done: 0 | visits: 143 | 2xx: 138 4xx: 4 5xx: 1
    ```

    - **terrains** — terrain groups that have come online so far
    - **alive** — lemmings currently navigating
    - **done** — lemmings that have completed their lifespan
    - **visits** — total page hits since the run started
    - **2xx / 4xx / 5xx** — response code class totals

    If `overflow` or `dropped` counts appear, your `-limit` is too low for the volume. See the [result collection concept](/concepts/result-collection) page.
  </Step>
  <Step title="Open the live dashboard">
    A one-time authentication token is printed before the boot summary. While the run is active, open `http://localhost:4000` in your browser and paste the token to unlock real-time counters and a live event stream.

    <Tip>
      The raw token is never stored after printing — only its SHA-512 hash is held in memory. If you miss it, start a new run.
    </Tip>
  </Step>
  <Step title="Read the final summary">
    When all lemmings have died, Lemmings prints a final summary and writes the report files:

    ```text
    ─────────────────────────────────────────
      lemmings v1.0.0 — final summary
    ─────────────────────────────────────────
      target:         http://localhost:8080/
      total lemmings: 100
      completed:      100
      failed:         0

      total visits:   1,247
      total bytes:    45.30 MB

      2xx:            1,231
      3xx:            8
      4xx:            6
      5xx:            2

      waiting room:   0 lemmings held
    ─────────────────────────────────────────
      report (md):   ./lemmings/localhost-8080/lemmings.2026.04.15.localhost-8080.md
      report (html): ./lemmings/localhost-8080/lemmings.2026.04.15.localhost-8080.html
    ```

    The HTML report includes per-path latency percentiles (p50, p90, p95, p99), response code breakdowns by URL, and waiting room statistics if applicable.
  </Step>
</Steps>

## Common run patterns

<Tabs>
  <Tab title="Quick test (no ramp)">
    For fast iteration, shrink the ramp to 0 and reduce terrain/pack:

    ```bash
    lemmings -hit http://localhost:8080/ -terrain 5 -pack 5 -until 15s -ramp 1s
    ```

    Wall clock: ~16 seconds.
  </Tab>
  <Tab title="Pre-launch simulation">
    Model traffic from 50 regions with 50 users each, ramping over 5 minutes:

    ```bash
    lemmings -hit https://staging.myapp.com \
        -terrain 50 -pack 50 -until 30s -ramp 5m
    ```

    Wall clock: ~5m30s. Total: 2,500 lemmings.
  </Tab>
  <Tab title="With crawl (no sitemap)">
    If your application doesn't have a sitemap, enable crawling:

    ```bash
    lemmings -hit https://myapp.com -terrain 10 -pack 10 -until 30s -crawl
    ```

    Lemmings crawls from the origin up to depth 3 and indexes all discovered URLs before starting.
  </Tab>
  <Tab title="CI mode">
    Disable TTY output and save the report to S3 for CI pipelines:

    ```bash
    lemmings -hit https://staging.myapp.com \
        -terrain 5 -pack 5 -until 10s \
        -tty=false \
        -save-to "s3://ci-reports/lemmings"
    ```
  </Tab>
</Tabs>
