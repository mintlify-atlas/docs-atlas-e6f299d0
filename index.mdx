Lemmings is a Go load-testing tool that simulates real users, not just HTTP requests. Each virtual user — called a lemming — has its own isolated session, navigates your site by following links, and lives for a configurable duration. The result is a load test that reveals how your application actually behaves when real people use it, not just when a script hammers one endpoint in a loop.

## Why Lemmings?

Most load tools fire dumb HTTP requests at a fixed rate. Lemmings does something different: it simulates real users. Each lemming lands on your site, picks a link from the indexed URL pool, follows it, dwells on the page, and keeps navigating until its time runs out. Cookie jars are isolated per lemming so session state never leaks between virtual users.

## Get started

<Steps>
  <Step title="Install Lemmings">
    Install with a single `go install` command — no dependencies to configure.

    ```bash
    go install github.com/andreimerlescu/lemmings@latest
    ```
  </Step>
  <Step title="Run a basic test">
    Point Lemmings at your application. It discovers URLs automatically from your sitemap and starts simulating traffic.

    ```bash
    lemmings -hit http://localhost:8080/ -terrain 10 -pack 10 -until 30s
    ```
  </Step>
  <Step title="Open the live dashboard">
    Lemmings prints a one-time authentication token at startup. Open `http://localhost:4000` in your browser and paste the token to see live metrics.
  </Step>
  <Step title="Read the report">
    When the run completes, Lemmings writes a Markdown and HTML report with per-path latency percentiles, response code breakdowns, and waiting room statistics.
  </Step>
</Steps>

<Note>
  Lemmings is designed for pre-production load testing. Run it against staging or a local environment — not against production systems you don't control.
</Note>
