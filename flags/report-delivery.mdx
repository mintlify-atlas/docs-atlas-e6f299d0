When a Lemmings run completes, it renders two report formats — a Markdown file and a self-contained HTML file — and delivers them to every destination you have configured. This page documents the `-save-to` flag, the five SMTP flags used for email delivery, and the environment variables that configure defaults for each.

---

## Flags

<ParamField path="-save-to" type="string" default=".">
  A comma-separated list of report delivery destinations. Each destination is
  parsed by its URI scheme and routed to the appropriate delivery target. All
  targets run concurrently after the run completes — a failure in one does not
  prevent others from receiving the report.

  **Alias:** `-s`

  **Accepted formats:**
  - Local path: `.`, `/var/reports`, `file:///var/reports`
  - S3 URI: `s3://bucket-name/optional/prefix`
  - Email: `mailto:address@example.com?subject=...`

  **Environment variable:** `LEMMINGS_SAVE_TO` — a comma-separated list of
  destinations that is merged with any value supplied via the flag. Duplicates
  are removed automatically.
</ParamField>

<ParamField path="-smtp-host" type="string" default="">
  The SMTP server hostname used when delivering to a `mailto:` destination.
  Overrides the `LEMMINGS_SMTP_HOST` environment variable when set.

  **Alias:** `-sh`

  **Environment variable default:** `LEMMINGS_SMTP_HOST` = `localhost`
</ParamField>

<ParamField path="-smtp-port" type="int" default="0">
  The SMTP server port. Overrides the `LEMMINGS_SMTP_PORT` environment variable
  when set to a non-zero value. TLS mode is auto-detected from the port number:
  `465` uses implicit TLS, `587` uses STARTTLS (with plain-SMTP fallback), and
  `25` uses plain SMTP.

  **Alias:** `-sp`

  **Validation:** must be between `0` and `65535` (inclusive). `0` means the flag
  is not set and the environment variable applies.

  **Environment variable default:** `LEMMINGS_SMTP_PORT` = `587`
</ParamField>

<ParamField path="-smtp-user" type="string" default="">
  The SMTP username for authentication. Overrides `LEMMINGS_SMTP_USER` when set.
  Leave empty for unauthenticated relay.

  **Alias:** `-su`

  **Environment variable default:** `LEMMINGS_SMTP_USER` = (empty)
</ParamField>

<ParamField path="-smtp-pass" type="string" default="">
  The SMTP password for authentication. Overrides `LEMMINGS_SMTP_PASS` when set.
  Leave empty for unauthenticated relay.

  **Alias:** `-spw`

  **Environment variable default:** `LEMMINGS_SMTP_PASS` = (empty)
</ParamField>

<ParamField path="-smtp-from" type="string" default="">
  The `From` address used in email delivery. Overrides `LEMMINGS_SMTP_FROM` when
  set.

  **Alias:** `-sf`

  **Environment variable default:** `LEMMINGS_SMTP_FROM` = `lemmings@localhost`
</ParamField>

## Delivery targets

### Local path

The default target. When `-save-to` is set to a local filesystem path (or left
at its default of `.`), Lemmings writes both report files under the path using
the following structure:

```
<save-to>/lemmings/<domain>/lemmings.YYYY.MM.DD.<domain>.md
<save-to>/lemmings/<domain>/lemmings.YYYY.MM.DD.<domain>.html
```

The directory tree is created automatically. If a file with the same name
already exists — which only occurs when Lemmings runs more than once against
the same target on the same calendar day — it is overwritten.

```bash
# Write to current directory (default)
lemmings -hit https://myapp.com -save-to .

# Write to an absolute path
lemmings -hit https://myapp.com -save-to /var/reports

# Explicit file:// URI
lemmings -hit https://myapp.com -save-to file:///var/reports
```

### S3

Uploads both the `.md` and `.html` report files to an S3 bucket with correct
`Content-Type` headers so they render correctly when accessed via S3 URLs or
CloudFront. The bucket must already exist — Lemmings does not create it.

```bash
lemmings -hit https://myapp.com -save-to s3://my-bucket/lemmings/reports
```

AWS credentials are resolved using the standard credential chain in order:

<Steps>
  <Step title="Environment variables">
    `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, and optionally
    `AWS_SESSION_TOKEN`.
  </Step>
  <Step title="Shared credentials file">
    `~/.aws/credentials` — the standard file written by `aws configure`.
  </Step>
  <Step title="IAM instance role">
    Works automatically on EC2, ECS, and Lambda with no additional configuration.
  </Step>
</Steps>

<Note>
  The bucket region is read from `AWS_DEFAULT_REGION` or `AWS_REGION`. The
  credentials must have `s3:PutObject` permission on the target bucket.
</Note>

### Email (`mailto:`)

Sends the HTML report inline as the email body and attaches the Markdown report
as a file. Uses standard RFC 6068 `mailto:` URI syntax for the address, subject,
and optional CC.

```bash
lemmings -hit https://myapp.com \
    -save-to "mailto:ops@mycompany.com?subject=Lemmings%20Results&cc=team@mycompany.com"
```

SMTP configuration is read from environment variables. Flag values override
environment variables when both are present:

| Environment variable | Default | Flag override |
|---|---|---|
| `LEMMINGS_SMTP_HOST` | `localhost` | `-smtp-host` |
| `LEMMINGS_SMTP_PORT` | `587` | `-smtp-port` |
| `LEMMINGS_SMTP_USER` | (empty) | `-smtp-user` |
| `LEMMINGS_SMTP_PASS` | (empty) | `-smtp-pass` |
| `LEMMINGS_SMTP_FROM` | `lemmings@localhost` | `-smtp-from` |

TLS mode is auto-detected from the port:

| Port | TLS mode |
|---|---|
| `465` | Implicit TLS |
| `587` | STARTTLS (falls back to plain if unsupported) |
| `25` | Plain SMTP |

## Multiple destinations

`-save-to` accepts a comma-separated list of destinations. All targets receive
the report concurrently. A failure in one target (unreachable S3 bucket, SMTP
connection error) is logged and does not prevent the remaining targets from
receiving the report.

```bash
lemmings -hit https://myapp.com \
    -save-to ".,s3://my-bucket/lemmings,mailto:ops@mycompany.com?subject=Lemmings%20Results"
```

You can also supply additional destinations via the `LEMMINGS_SAVE_TO`
environment variable. Its value is a comma-separated list that is merged with
the `-save-to` flag value before the run starts. Duplicates are deduplicated
automatically.

```bash
# Flag provides local and S3; env var adds email
LEMMINGS_SAVE_TO="mailto:ops@mycompany.com?subject=Results" \
    lemmings -hit https://myapp.com -save-to ".,s3://my-bucket/lemmings"
```

## Full example with SMTP overrides

```bash
lemmings -hit https://myapp.com \
    -save-to "mailto:ops@mycompany.com" \
    -smtp-host mail.mycompany.com \
    -smtp-port 587 \
    -smtp-user sender@mycompany.com \
    -smtp-pass "$SMTP_PASSWORD" \
    -smtp-from noreply@mycompany.com
```

<Warning>
  Avoid passing SMTP credentials directly on the command line in shared or
  logged environments. Use environment variables (`LEMMINGS_SMTP_USER`,
  `LEMMINGS_SMTP_PASS`) or a secrets manager instead.
</Warning>
